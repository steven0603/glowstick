## 一、系統概覽

### 1.1 系統定位

GlowStick 是一套專為「**投資競賽記錄與追蹤**」設計的本地端基金管理系統。它模擬一間真實的證券投資信託基金公司的日常作業，自動從網路抓取市場資料、計算每日淨值（NAV）、記錄所有交易、並產出競賽所需的各種報告。

整套系統在你的電腦本機執行，所有資料儲存在 `portfolio.db` 這個資料庫檔案中，不需要網路伺服器，用指令列（Terminal）操作即可。

### 1.2 競賽基本參數

| 項目 | 設定值 |
|------|--------|
| 基金名稱 | GlowStick X 證券投資信託基金 |
| 管理公司 | 空間叉叉投資信託(股)公司 |
| 初始資金 | 1 億美元（USD 100,000,000） |
| 競賽起始日 | 2026-03-23（首個交易日） |
| 競賽結算日 | 2026-05-29 |
| 績效基準 | 台灣加權股價指數（TAIEX，^TWII） |
| 計價幣別 | 美元（USD）為主，台幣（TWD）為輔 |
| 無風險日利率 | 0.003% |

### 1.3 投資限制規範

根據基金公開說明書，系統會自動偵測以下違規：

| 限制項目 | 規定 |
|----------|------|
| 現金比例上限 | 不得超過總淨值的 30% |
| 單一標的比例上限 | 不得超過總淨值的 30% |
| 持有標的種類 | 最少 10 種、最多 20 種 |

### 1.4 系統啟動方式

```
Terminal 進入 glowstick 目錄後執行：
python3 main.py
```

啟動後，系統會自動執行「補齊作業」（詳見第八節），接著顯示最新淨值快照，再進入功能選單。

## 二、核心設計邏輯

### 2.1 整體資料流程

```
Yahoo Finance（網路）
        ↓
  data_fetcher.py（抓取）
        ↓
  portfolio.db（本地資料庫）
        ↓
  portfolio.py（計算 NAV）
        ↓
  reporter.py（輸出報告）
```

所有市場資料（股價、匯率、TAIEX）皆透過 **Yahoo Finance** 的免費 API 抓取。抓下來的資料存進本地資料庫，之後所有計算都從資料庫讀取，不需要重新連網。

### 2.2 淨值（NAV）計算邏輯

基金每日淨值的計算流程如下：

```
步驟 1：確認當日持倉（從初始持倉 + 所有交易紀錄累積推算）
步驟 2：取得各持股當日收盤價（找不到則沿用最近一筆，即 forward fill）
步驟 3：計算各股市值 = 持股數 × 收盤價（台幣）
步驟 4：加總所有股票市值 + 現金餘額 = 台幣總淨值
步驟 5：台幣總淨值 ÷ 當日匯率 = 美元淨值（NAV USD）
```

**重要**：NAV 以美元計算是因為競賽規則以美元為比較基準，但所有交易以台幣執行。

### 2.3 持倉追蹤方式（Trade Replay）

系統不直接儲存「現在持有多少股」，而是採用**交易重播法**：

- 每次需要知道某日的持倉，系統會從初始資金出發
- 按照時間順序把所有交易一筆一筆重新跑過
- 算到指定日期就停下來，那個時刻的狀態就是當日持倉

這個設計的好處是：**歷史上任何一天的持倉都可以精確重算**，不會因為中間有更動而出錯。

### 2.4 匯率處理

系統使用 Yahoo Finance 的 `TWD=X` ticker 取得台幣兌美元匯率。

- 交易日有新匯率就存入資料庫
- 週末或假日沒有新匯率，自動沿用最近一個交易日的匯率（forward fill）
- TAIEX 指數也同樣轉換為美元計價，才能和基金 NAV 做蘋果對蘋果比較

## 三、資料庫結構

資料庫為單一檔案 `portfolio.db`（SQLite 格式），共有 10 張資料表：

### 3.1 資料表一覽

| 資料表名稱 | 用途 | 主要欄位 |
|-----------|------|---------|
| `holdings` | 目前持股快照（不作為計算依據） | name, ticker, shares, avg_cost_twd |
| `cash` | 現金餘額（只有 1 筆記錄） | amount_twd |
| `price_history` | 各股每日 OHLCV | date, ticker, open, high, low, close, volume |
| `exchange_rate_history` | 每日台美匯率 | date, rate |
| `nav_history` | 每日基金淨值 | date, nav_twd, nav_usd, exchange_rate |
| `taiex_history` | 每日 TAIEX 指數（含美元換算） | date, close, close_usd |
| `orders` | 委託單記錄（含已執行、已取消） | id, submitted_date, ticker, direction, quantity, status... |
| `trade_log` | 已成交交易紀錄 | id, date, ticker, direction, quantity, price_twd, fee_twd, net_twd |
| `system_state` | 系統設定鍵值對 | key, value |
| `sqlite_sequence` | SQLite 內部用 | — |

### 3.2 `system_state` 重要鍵值

| Key | 用途 |
|-----|------|
| `initialized` | 是否已完成初始化（true/false） |
| `initial_snapshot` | 系統起始狀態（初始資金、空持倉） |
| `initial_trades_added` | 3/23 初始買進是否已寫入 trade_log |

### 3.3 資料流向圖

```
orders（下單）
    ↓ process_pending_orders()
trade_log（成交紀錄）
    ↓ _get_holdings_and_cash_at_date()
    ↓ Trade Replay
holdings（當日持倉）+ cash（現金）
    ↓ calculate_nav()
nav_history（每日 NAV）
```

### 功能 1：更新今日價格 & 計算淨值

**做什麼**：抓取指定日期的市場資料，執行當日的待執行委託，並計算並儲存當日的基金淨值。

**操作流程**：
1. 輸入要更新的日期（直接 Enter 預設為今天）
2. 系統依序：
   - 抓取當日台美匯率
   - 抓取當日 TAIEX 指數
   - 抓取所有持股當日 OHLCV 資料
   - 執行當日所有 PENDING 委託（市價單用開盤價，現價單依觸價條件）
   - 計算並儲存當日 NAV

**輸出資訊**：
- 當日匯率
- TAIEX 指數
- 基金台幣及美元淨值
- 累積報酬率

**注意**：每個交易日都應執行此功能，確保資料完整。

## 五、下單系統規則

### 5.1 委託執行時間

- **送出時間**：任何一天皆可送出委託（包含週末、假日）
- **執行時間**：次一個「有市場資料」的交易日

> 例：週日 4/19 送出委託 → 4/20（週一）執行

### 5.2 市價單（MARKET）

- 無條件成交，**以執行日開盤價為成交價**
- 若當日無開盤價資料（系統會嘗試重新抓取），則跳過等待下一次更新

### 5.3 現價單（LIMIT）

- 買進現價單：當日最低價 ≤ 限價，以**限價**成交
- 賣出現價單：當日最高價 ≥ 限價，以**限價**成交
- 若當日未觸及限價，委託自動**取消**（不保留到下一交易日）

### 5.4 費用計算

| 項目 | 費率 | 說明 |
|------|------|------|
| 買進手續費 | 0.03705% | 計算基礎：成交價 × 股數 |
| 賣出手續費 | 0.03705% | 計算基礎：成交價 × 股數 |
| 賣出交易稅 | 0.3% | 計算基礎：成交價 × 股數 |

**買進淨支出** = 成交價 × 股數 × (1 + 0.03705%)

**賣出淨收入** = 成交價 × 股數 × (1 − 0.03705% − 0.3%)

### 5.5 持倉計算規則

- **平均成本（avg_cost_twd）**：以成交價為基準，不含手續費
  - 買進：加權平均 = (舊持倉數×舊均成本 + 新買數×成交價) / (舊持倉數 + 新買數)
  - 賣出：均成本不變

## 六、費用與計算公式

### 6.1 買進

```
手續費   = 成交價 × 股數 × 0.0003705
淨支出   = 成交價 × 股數 + 手續費
         = 成交價 × 股數 × 1.0003705
現金變動 = −淨支出（現金減少）
```

### 6.2 賣出

```
手續費+稅 = 成交價 × 股數 × (0.0003705 + 0.003)
          = 成交價 × 股數 × 0.0033705
淨收入    = 成交價 × 股數 − 手續費+稅
          = 成交價 × 股數 × 0.9966295
現金變動  = +淨收入（現金增加）
```

### 6.3 NAV 計算

```
各股市值(TWD) = 持股數 × 當日收盤價
總股票市值    = Σ 各股市值
台幣 NAV     = 總股票市值 + 現金餘額(TWD)
美元 NAV     = 台幣 NAV ÷ 當日匯率(TWD/USD)
```

## 七、Alpha 計算方法

### 7.1 公式定義

競賽採用標準 **CAPM Alpha** 計算：

| 符號 | 定義 |
|------|------|
| T | 共同交易日總數 |
| r_p,t | 基金第 t 日美元報酬率 |
| r_m,t | TAIEX 第 t 日美元報酬率 |
| r̄_p | 基金平均日報酬率 = Σr_p,t / T |
| r̄_m | TAIEX 平均日報酬率 = Σr_m,t / T |
| r_f | 無風險日利率 = 0.003% |

### 7.2 計算步驟

**步驟 1：計算平均日報酬率**
```
r̄_p = Σ r_p,t / T
r̄_m = Σ r_m,t / T
```

**步驟 2：計算 Beta（系統性風險）**
```
β_p = Σ(r_p,t − r̄_p)(r_m,t − r̄_m)
      ─────────────────────────────────
             Σ(r_m,t − r̄_m)²
```

**步驟 3：計算 Alpha（超額報酬）**
```
α_p = r̄_p − [r_f + β_p × (r̄_m − r_f)]
```

### 7.3 重要說明

- 報酬率序列以**美元計價**，排除匯率影響
- TAIEX 原為台幣指數，先除以當日匯率轉換為美元再計算報酬率

**報酬率計算基準（t=1 的 t-1 基準）：**

| 序列 | 第一個交易日的 t-1 基準 |
|------|----------------------|
| 基金 r_p | 初始資金 `INITIAL_CAPITAL_USD`（1 億美元），而非首日 NAV |
| TAIEX r_m | 競賽起始日（START_DATE）**前一個交易日**的 `close_usd` |

這樣設計確保基金和 TAIEX 的報酬率都從競賽「起跑點」開始計算，不會因為 3/23 當天的漲跌導致基準偏移。

- Alpha 為**日均值**，年化僅為參考（× 252 交易日）
- 評分時，同樣的 Alpha 越高排名越前（具體評分公式依競賽規定）

## 八、自動補齊機制

### 8.1 機制說明

每次啟動系統（`python3 main.py`），系統會自動檢查：

```
1. 從資料庫取出最後一筆 NAV 的日期（last_nav_date）
2. 若 last_nav_date < 今天，自動補齊中間所有日期
3. 補齊範圍：(last_nav_date + 1天) 到 今天
```

### 8.2 補齊作業順序

```
步驟 1：批次從 Yahoo Finance 抓取補齊期間的所有資料
        → 匯率（TWD=X）
        → TAIEX（^TWII）
        → 所有持股 OHLCV

步驟 2：逐日處理（從最早補齊日到今天，每天都跑）
        → 執行當日 PENDING 委託
        → 計算並儲存當日 NAV
```

### 8.3 週末與假日處理

- 週末和假日**不會有股價或匯率資料**，屬正常現象
- NAV 仍會計算（因為基金每日都有淨值），但股價使用最近交易日資料（forward fill）
- 現價單在假日不觸價，市價單在假日也不執行，等到下一個有資料的交易日才處理

### 8.4 已是最新時的行為

```
若 last_nav_date >= 今天
→ 顯示「資料已是最新，無需補齊」
→ 直接跳到顯示快照
```

## 九、操作守則

### 9.1 日常操作流程（每個交易日）

```
每天市場收盤後（台灣時間 13:30 後）執行：

1. 啟動系統（python3 main.py）
   → 系統自動補齊到今天
   → 查看啟動快照確認淨值正確

2. 若有委託要下：
   → 選功能 7 下委託單
   → 當天收盤後送出的委託，次一交易日開盤執行

3. 確認一切無誤後：
   → 選功能 4 生成 Markdown 並複製到競賽系統

4. 定期選功能 3 查看績效走勢
5. 定期選功能 6 計算最新 Alpha
```

### 9.2 下單前注意事項

1. **確認 Ticker 正確**：台灣上市股票格式為 `XXXX.TW`，上櫃為 `XXXX.TWO`，輸入前請務必確認
2. **股數必須為整數**：現實最少購買 1 股，不接受小數點
3. **賣出前確認持倉**：可先用功能 2 查看當前持股數量，避免超賣
4. **現價單當日有效**：現價單若未觸價，隔日自動取消，需重新送出

### 9.3 投資限制自查

每次下單前，建議：
1. 功能 2 查看持倉比例
2. 確認新買入後單一標的不超過 30%
3. 確認現金比例不超過 30%
4. 確認持有標的種類在 10～20 種之間

系統會在功能 2 的表格底部自動提示違規，但**不會阻擋下單**，操作人員需自行把關。

### 9.4 資料異常的處置方式

| 異常狀況 | 處置方式 |
|---------|---------|
| 某日價格明顯異常（週末出現資料） | 直接在 Terminal 執行 SQL 刪除錯誤資料，再重算 NAV |
| 委託日期執行錯誤 | 直接修改 trade_log 和 orders 表的 date 欄位，重算相關 NAV |
| Ticker 輸入錯誤 | 修改 trade_log 和 orders 的 ticker 欄位，補充正確股票的 price_history，重算 NAV |
| 費率設定需變更 | 修改 config.py，再執行批次更新腳本更新所有歷史 fee_twd / net_twd |
| 資料嚴重損毀 | 選功能 0 重新初始化（注意：會清除所有手動調整） |

### 9.5 時間規範

- 系統所有時間以**台北時間（UTC+8）**為準
- 交易日以台灣證券交易所的開休市日曆為準
- Yahoo Finance 抓取的台股資料日期已是台灣時間，不需轉換

## 十、常見問題排除

### Q1：啟動後顯示「資料已是最新」但淨值看起來不對？

可能原因：
- 收盤後資料尚未更新（通常 14:00 後 Yahoo Finance 才會有當日完整資料）
- 選功能 1，手動輸入今天日期強制重新抓取

---

### Q2：下委託單後隔天沒有自動執行？

可能原因：
1. 市價單：當日無開盤價資料 → 等待下次啟動系統或功能 1 更新
2. 現價單：當日未觸價 → 系統已自動取消，需重新送出

確認方法：選功能 8 查看待執行委託，若該筆已不見，表示已執行或取消；選功能 9 查看交易紀錄確認。

---

### Q3：某標的的收盤價顯示 0.00 或不更新？

可能原因：
- Yahoo Finance 對該 Ticker 暫時無資料
- Ticker 格式錯誤（確認上市用 `.TW`，上櫃用 `.TWO`）

處置：可在 Terminal 直接測試：
```python
import yfinance as yf
df = yf.download("XXXX.TW", period="5d")
print(df)
```

---

### Q4：Markdown 報告顯示已賣出的標的？

已透過「浮點數精度修正」解決（股數殘差 < 0.0001 視為已清空）。若仍發生，請回報。

---

### Q5：Alpha 計算顯示「資料不足」？

原因：需至少 1 個共同交易日（基金 NAV 和 TAIEX 都有資料的日子）。此外，TAIEX 需要有競賽起始日前一交易日的資料作為計算基準；若該資料缺漏，Beta/Alpha 計算會失準。
解決：先執行功能 1 補齊歷史資料，確認功能 3 有顯示資料後再試。若 TAIEX 基準日（3/22 前）資料異常，可執行功能 b 以台銀匯率重建。

---

### Q6：如何手動修改單筆歷史交易？

資料庫使用 SQLite，可用以下工具直接操作：
- **DB Browser for SQLite**（圖形介面，推薦）：https://sqlitebrowser.org
- 或在 Terminal 執行 `sqlite3 portfolio.db` 後輸入 SQL 指令

修改完成後，需用 Python 腳本重算受影響日期的 NAV：
```python
import sys; sys.path.insert(0, '.')
import portfolio as pf
pf.save_nav_for_date('2026-04-20')  # 替換為受影響的日期
```
